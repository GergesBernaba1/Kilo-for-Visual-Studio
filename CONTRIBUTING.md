# Contributing to Kilo for Visual Studio

Thank you for contributing! Please follow these steps:

1. Fork the repository and create branches from `main`.
2. Open an issue before starting large changes.
3. Build and run tests locally:
   - `dotnet restore`
   - `dotnet test Kilo.VisualStudio.Tests/Kilo.VisualStudio.Tests.csproj`
4. Follow code style and naming conventions.
5. Add/Update tests for new behavior.
6. Submit pull request with a summary of changes and test results.

## Communication
- Use issues for bug reports and feature requests.
- Mention `@GergesBernaba1` for high-priority review.
